🔴 Pain Points của sinh viên

⏰ "Deadline hôm nay là mấy giờ?"
Phải check LMS + Discord + hỏi bạn bè mới biết deadline chính xác. Có khi deadline đã thay đổi mà không hay.
⚠️ Rủi ro: nộp trễ vì thông tin cũ
📄 "Slide phiên bản mới nhất ở đâu?"
Drive có nhiều folder, PDF cũ lẫn mới. Không biết file nào là bản cuối cùng.
⚠️ Rủi ro: học nhầm tài liệu cũ, làm sai yêu cầu
🐙 "Fork repo nào? Submit branch nào?"
GitHub có nhiều repo, không biết đâu là starter code chính thức của lab này.
⚠️ Rủi ro: fork nhầm repo, submit sai branch
📢 "TA vừa thông báo gì mới không?"
Discord có hàng trăm message mỗi ngày. Thông báo quan trọng bị trôi giữa chat.
⚠️ Rủi ro: bỏ lỡ update quan trọng từ instructor
🎯 "Tôi cần làm gì để đạt điểm cao?"
Rubric nằm trong PDF + LMS + SCORING.md trên GitHub. Phải đọc nhiều chỗ để tổng hợp.
⚠️ Rủi ro: nộp thiếu phần quan trọng, mất điểm không đáng
🕐 Kết quả: Mất 15–30 phút/ngày chỉ để TÌM THÔNG TIN thay vì tập trung CODE

🔄 ReAct Loop — Kỹ thuật cốt lõi

💭 Thought → ⚡ Action → 👁 Observation → ... → ✅ Final Answer
👤 Student: "Lab 3 nộp khi nào và repo starter ở đâu?"
💭 Thought: Câu hỏi có 2 phần — deadline (LMS) và repo (GitHub). Tìm LMS trước.
⚡ Action: lms_tool("deadline lab 3")
👁 Observation: [LMS] Lab 3 deadline: 01/06/2026 23:59. Submit: lms.aivn.vn/lab3
💭 Thought: Đã có deadline. Tiếp theo tìm repo trên GitHub.
⚡ Action: github_tool("starter repo lab 3")
👁 Observation: [GitHub] Repo: github.com/aivn-course/lab3 | Branch: main | Fork để submit.
💭 Thought: Đã đủ thông tin. Tổng hợp câu trả lời.
✅ Final Answer: Lab 3 nộp trước 23:59 ngày 01/06. Repo starter tại github.com/aivn-course/lab3, fork về và submit link trên LMS.
